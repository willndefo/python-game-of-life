# TP : Le jeu de la vie de Conway

Le but de ce TP est de réaliser une implémentation du "jeu de la vie" imaginé par John Conway.

[Vidéo : le jeu de la vie](https://www.youtube.com/watch?v=S-W0NX97DB0)

Le jeu de la vie est en réalité bien plus un algorithme qu'un jeu. Pour être exact, il s'agit d'un automate cellulaire (il en existe d'autres !).

Créez un fork de ce repository et clonez-le.

Installez le module `pygame` grâce à la commande suivante (et en adaptant à votre version de python):

```

```

Pour vérifier la bonne installation, vous pouvez lancez un des exemples inclus dans le module :


## Étape 1 : pygame

Dans cette étape, nous allons aborder tout ce que vous avez besoin de savoir sur `pygame` pour implémenter le jeu de la vie. Nous avons simplement besoin de savoir :

- lancer le jeu
- gérer un cycle de vie (un cycle de rendu dans le jeu)
- dessiner des carrés (qui seront les cellules)

### a. Lancer le jeu

L'instruction suivante permet d'ouvrir une fenêtre de jeu :

```py
screen: pygame.surface.Surface = pygame.display.set_mode((1000, 1000))
```

`(1000, 1000)` est un [tuple](https://courspython.com/tuple.html) représentant la taille de l'écran souhaitée.

_Note : une bonne pratique est de stocker ce genre de valeur dans un fichier de constantes. Elles seront ainsi toutes aux même endroit et cela vous permettra de modifier facilement les paramètres de votre simulation._

En lançant votre programme, la fenêtre devrait s'ouvrir et se fermer immédiatement.

### b. Le cycle de vie

Voici comment demander au jeu de rester actif et de se mettre à jour une fois par seconde.

```py
clock = pygame.time.Clock()

done: bool = False

# While the game is not over
while not done:

    # Listen for all events
    for event in pygame.event.get():

        # Quit the infinite loop when the user presses the close button
        if event.type == pygame.QUIT:
            done = True

    print("Update !")
    clock.tick(1)

pygame.quit()
```

La fenêtre doit maintenant rester affichée, et vous devirez voir le message `Update !` à chaque seconde. Vous pouvez diminuer le `tick` pour augmenter la fréquence de mise à jour.

Prenez le temps de comprendre l'ensemble du code. Vous pouvez le ré-utiliser pour tous les programmes et jeux de la sorte.

### c. Les carrés

À chaque itération, nous devrons commencer par effacer la totalité de la fenêtre :

```py
screen.fill((0, 0, 0))
```

Comme vous pouvez le constater, la couleur noire correspond au tuple `(0, 0, 0)`.

Et voici comment dessiner un carré blanc :

```py
pygame.draw.rect(screen, (255, 255, 255), (0, 0, 100, 100))
pygame.display.flip()
```

Les quatres derniers chiffres de la fonction `rect()` sont les coordonnées du carré : x, y, largeur, hauteur.

La fonction `flip` permet de rendre effectives les modifications effectuées sur l'écran. Il faut l'appeler à la fin de chaque update. Si vous dessinez plusieurs carrés à la suite, appellez là en dernier !

Entraînez-vous à placer des carrés sur l'écran.

Vous savez tout !

## Étape 3 : calculer l'état suivant

Le coeur de l'algorithme est le calcul de l'état suivant à partir des règles du jeu de la vie.

Par exemple, l'état présenté à la partie suivante évolue vers la forme suivante :

```py
[
    [0, 1, 0],
    [0, 1, 0],
    [0, 1, 0]
]
```

Puis il revient à l'état initial (c'est un oscillateur).

Implémentez une fonction retournant un nouvel état en fonction d'un état donné.

**Attention** : au début de votre fonction, vous devrez faire une *copie profonde* de votre état actuel avant de le modifier. Sinon, vous modifierez l'état alors même que vous êtes en train de l'analyser, et l'algorithme ne fonctionnera pas. Voici comment faire une copie profonde :

```py
import copy
new_array = copy.deepcopy(array)
```

Le fichier `test.py` contient un test qui vous permettra de savoir si l'algorithme est correctement implémenté.

Vous n'avez plus qu'à générer un nouveau rendu avec votre nouvel état !

Testez votre programme avec différentes structures dont vous trouverez les modèles sur Internet.

Vous pouvez à cette étape rendre votre TP grâce à une pull request vers le repository original.
